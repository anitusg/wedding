Happy Wedding

This snowflake animation uses only CSS and SVGs — my second try at this concept.

Tiling a graphic in SVG (using pattern, feTile, and friends) can cause performance issues, so instead, the SVGs here are tiled via CSS: they're generated by SCSS as data: URIs, and then used as background-images. Different parallax layers are animated semi-independently by using multiple offsets on background-position. On desktop, at least, this seems to have completely alleviated the relevant performance issues, even when resizing the viewport.

On browsers that support animation-composition, three basic effects are applied: downward movement; lateral movement; and a horizontal "wobble." Each effect has an independently configurable speed, and the wobble magnitude is also configurable. On browsers without animation-composition (i.e. Firefox, as of this writing), we use SCSS to compute the least common multiple of the various animation timers, and then construct a single merged animation. Unfortunately, the wobble animation is not enabled for those browsers, as merging it into the other animations would (due to its use of a cubic Bezier easing function) be prohibitively difficult.

Potential room for improvement includes making the distant snowflakes more dense. This could be accomplished by adding additional layers and varying their animation timings, but varying those timings is rather difficult without animation-composition, and that's an effect that I would rather have working cross-browser. (With animation-composition, we could just have our baseline animations not even move the n-th layer(s), and then define secondary animations that move only those layers, and give those secondary animations an animation-delay. We already use that technique to desynchronize the wobble animations between layers.)

It may also be possible to optimize the distance fog that is applied to snowflakes. Currently, this effect works by specifying multiple near-identical SVGs that use different snowflake colors. This requires duplicating the full content of the SVGs, because CSS string interpolation does not work on url() values. However, I suspect we may be able to reuse a single SVG (stuffed into a CSS variable) by using multiple values on background-blend-mode; we could potentially place a solid-color image-type background (generated with linear-gradient) between the different SVG layers, with the fog color at zero alpha, and then take advantage of whether and how the blending algorithms use and preserve alpha values (I'd have to consult the spec to check that).

Another potential effect to explore would be a super dense and barely visible layer of snowflakes behind the page content, using either a separate pseudo-element or the html element directly.
